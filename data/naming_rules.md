# Naming Rules Summary

## 기본 패턴
```
법무실_사건명(프로젝트명)_문서명_날짜
```

## 사건명(프로젝트명) 결정 규칙
- **폴더 트리의 3번째 레벨** 폴더명을 사용 (LLM/OCR 분석 결과 무시)
- `root_folder` 기준 `parts[1]` (0-indexed)
- 폴더명 앞의 `[검토수행]`, `[검토종결]` 등 괄호 태그는 **자동 제거**

```
법무DB               ← root_folder (제외)
  소송 및 중재        ← parts[0] = 대분류 (파일명에 미사용)
  [검토수행] 두바이 Sunrise 소송  ← parts[1] = 사건명 ★ 태그 제거 후 사용
    1. 사건기록       ← parts[2] = 소분류 (파일명에 미사용)
      파일.pdf
```

결과 파일명: `법무실_두바이 Sunrise 소송_법률통지_Legal Notice to SUNRISE_140519.pdf`
(태그 `[검토수행]` 제거됨)

## 문서명 결정 규칙
- LLM document_title 우선 (예: 가압류결정, 계약위반의견서, 세관심리결과)
- 없으면 doc_type 키워드 매핑 (예: 소송진행보고, 법률의견)
- 없으면 요약에서 추출

## 날짜 형식
- `YYMMDD` (예: 140519)
- 파일명 또는 문서 본문에서 추출
- 추출 실패 시 `date_unknown` → 수동 검토 대상

## 수동 검토 대상
- 날짜 불명 (`date_unknown`)
- 텍스트 추출 실패
- 신뢰도 < 0.78
- 중복 파일명 충돌
